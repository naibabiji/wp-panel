# 更新说明

## v1.2.12

### 修复
- 修复停用/启用站点时 nginx enabled 路径使用域名拼接文件名，导致配置文件名与域名不一致的站点操作失败的问题；现在改为使用实际存储的 nginx 配置文件名
- 增强站点删除、暂停、启用、重装的路径安全校验，避免异常历史数据或路径污染导致误删托管目录外的文件
- 修复暂停/启用站点时 Nginx 重载失败可能导致数据库状态与实际 enabled 软链接状态不一致的问题，失败时会尽量恢复原软链接状态
- 优化 WordPress 重装流程，先完成临时目录部署校验，再清理旧站点目录和重建数据库，降低安装包不可用、解压失败等前置错误造成半成品站点的风险
- 优化 WordPress 重装失败提示，页面展示安全的失败阶段原因，完整底层错误仍保留在服务端日志中
