# 更新说明

## 下个版本

### 修复：大型数据库上传恢复稳定性

- 数据库恢复改为后台任务执行，避免大 SQL 包导入时后台页面长期卡在单个 HTTP 请求中。
- 上传恢复和备份恢复接口继续保留兼容响应，同时返回任务 ID，前端会轮询恢复状态并显示成功或失败原因。
- 恢复前增加 SQL 包预检：确认 `.sql`、`.sql.gz`、`.zip` 可读取，并在清空目标库前拦截明显无效或高风险 SQL。
- SQL 预检会识别字符串内容，文章或数据字段中包含 SQL 教程代码时不会误判为跨库语句。
- 导入时会自动跳过字符串外的 `CREATE DATABASE`、`DROP DATABASE`、`USE` 和 `DEFINER` 语句，兼容更多常见备份工具导出的 SQL 包。
- 导入期间自动关闭外键检查，兼容子表先于父表创建的备份文件，导入结束后恢复外键检查。
- 支持 WordPress 网站和通用 PHP 网站数据库包，不强制要求 WordPress 表结构。
- 改进 gzip 和 mysql 导入失败信息，失败时更容易定位原因。

### 修复：登录错误提示

- 登录请求遇到网络层失败时，不再直接显示浏览器英文 `Load failed`，改为中文提示用户检查面板服务、网络、HTTPS 证书或访问入口。
- 登录接口返回用户名或密码错误时，前端会显示后端返回的明确原因，不再被统一会话失效跳转吞掉。

### 配套插件

- WP Panel Optimizer 设置页新增“清除 Nginx 缓存”按钮，方便移动端后台手动清理缓存。
- WP Panel Optimizer 版本更新为 1.1.5，并同步插件 readme 更新记录。

### 安全与可追溯性

- 上传恢复临时文件仅允许 `/tmp` 路径，并在任务结束后清理。
- 拒绝包含 `CREATE DATABASE`、`DROP DATABASE`、`USE`、`DEFINER=` 的恢复包，降低跨库恢复和权限污染风险。
- 恢复任务状态接口会校验任务类型和站点 ID，避免跨站点查询任务结果。
- 操作日志补充数据库备份和恢复任务的站点域名，便于排查恢复失败或误操作。

### 测试

- 增加恢复文件预检测试，覆盖通用 SQL、WordPress SQL、gzip、大 INSERT 行和危险 SQL 变体。
- 增加恢复任务状态查询、队列状态轮询和操作日志 target 的回归测试。
