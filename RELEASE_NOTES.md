## 新增

- **扫描防御**：面板 8443 端口自动拦截非浏览器请求（curl、脚本、扫描器），直接写入 Nftables 丢弃数据包，封禁 30 天
- **安全测试邀请**：README 新增白帽测试说明，GitHub Security 标签页已启用私下漏洞报告

## 修复

- **路径穿越漏洞**：文件管理器路径校验增加尾部斜杠，防止相邻目录名绕过（P1-01）
- **BasicAuth 封禁失效**：时区 Bug 导致 BasicAuth 连续失败永不封禁，已修正为 UTC 统一时间
- **面板认证封禁**：Web 登录和 BasicAuth 失败封禁改为 Nftables 持久规则，不再依赖 fail2ban（消除 bantime 不一致问题）
- **过期封禁清理**：到期封禁自动从列表移除并清理 Nftables 规则
- **历史封禁记录**：修复永远为空的 Bug
- **错误信息泄露**：全项目 ~65 处 API 错误返回改为通用文案，内部路径/命令输出仅记录服务器日志
- **配置文件泄露**：Nginx 模板新增 `wp-panel-config.json` 拦截，防止插件 API Key 被公网访问

## 依赖升级

- lego v4.17.4 → v4.25.2（ACME HTTPS 校验漏洞）
- x/net v0.26.0 → v0.55.0（IDNA 域名处理漏洞）
- Go 最低要求：1.22 → 1.25.0

## 变更

- 封禁等级扩展为 5 级：限速 / 临时10分钟 / 临时24小时 / 临时30天 / 永久封禁
- Nftables 持久集统一管理扫描防御、面板认证、永久封禁三类封禁
- 防御规则说明文档更新，增加扫描防御和服务器验证命令
