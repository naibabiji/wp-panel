## v1.2.0-rc2

### Bug 修复
- **修复跨域名数据库名冲突**：创建网站时，`buildSiteName` 错误丢弃 TLD 后缀（如 `.com`、`.cn`），导致 `test.com` 和 `test.cn` 生成相同的数据库名 `db_test`。现已修复，不同 TLD 将生成不同的数据库名（如 `db_test_com` vs `db_test_cn`）。
- **SSL 证书申请失败时显示具体原因**：之前申请 Let's Encrypt 证书失败仅返回通用提示，真正错误原因（DNS 未解析、频率限制、验证文件不可访问等）只记录在服务器日志。现在将具体错误信息返回到前端，方便定位问题。
- **修复解压 zip 包时误报"非法路径"**：当压缩包包含多级目录（如 `astra-addon/addons/`）时，`resolvePathForAccess` 仅检查一级父目录的符号链接，父目录未创建时误判为非法路径。改为向上逐级查找已存在的目录，不再误报。
- **修复 `wp password` 命令两个问题**：(1) SQLite 更新未检查 `db.Exec` 错误，若写入失败用户拿到密码但实际未存入数据库，现增加错误检查；(2) 每次 CLI 子命令都触发全站 Nginx/FPM 重建，重置密码却遍历全站无意义操作，现改为仅在面板启动时执行。
- **修复勾选告警规则时误报 SMTP 配置错误**：`alertManager` 组件获取设置时将 SMTP 配置与告警规则混存在同一对象中，勾选规则保存时 SMTP 值被转为 true/false 字符串，触发后端校验"SMTP 端口不正确"、"SMTP 加密方式不正确"。改为只存储和发送告警规则字段。
